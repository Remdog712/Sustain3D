In Practice: TREC Cooper Hawk Case Study

This page documents how the Sustain 3D workflow was applied in practice for the Tom Ridge Environmental Center (TREC) Cooper Hawk specimen.

1) Project context

  • Project: TREC Specimen Scans (TREC_SpecimenScans_2026)
  • Case focus: Cooper Hawk specimen (TREC_0001_CooperHawk)
  • Capture hardware: Artec Space Spider
  • Processing software: Artec Studio 17 Professional, then Blender for access optimization
  • Naming basis: VAR Lab naming pattern TREC_XXXX_<SpecimenName>

2) How the workflow was handled

  1. TREC provided the specimen model for scanning.
  2. Raw scans and exported high-resolution OBJ were created from Artec Space Spider capture.
  3. Raw and preservation packages were retained, and a smaller web-access model was derived.
  4. Blender working files were kept in project processing folders.
  5. Access GLB was published to Sketchfab.
  6. Preservation master package and metadata were copied into repository preservation storage for long-term retention.
