Glossary of core 3D and preservation terms

3D model: A digital representation of shape (geometry), often with texture, material, and scale information.
Checksum: A hash value (for example SHA-256) used to confirm a file has not changed or become corrupted.
Repository: The organized storage and management location for project files, metadata, and documentation.
File structure: The folder hierarchy and naming pattern used so files can be found, linked, and preserved reliably.
Metadata: Structured descriptive and technical information about an object, scan event, or file.
Paradata: Documentation of process decisions, tools, and assumptions made during 3D creation and editing.
Point cloud: A set of XYZ coordinate points captured from scanning before a mesh is created.
Mesh: Connected vertices, edges, and faces that define the surface of a 3D object.
Texture: Image data mapped to a mesh to add color or surface detail.
Preservation master: Highest-quality retained version intended for long-term maintenance and future reuse.
Access derivative: A lower-weight copy made for delivery (web, teaching, outreach), not long-term preservation.
Fixity check: Routine verification that stored files still match their original checksums.

Quick implementation reminders

  • Keep preservation masters separate from access copies.
  • Record storage location and checksum for each critical file set.
  • Use consistent IDs across folder names, inventory rows, and published records.
