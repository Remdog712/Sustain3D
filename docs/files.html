Goal

Use predictable names and folder patterns so 3D packages stay connected and understandable.

Where this fits

This step comes after repository setup and before metadata detail entry.

Recommended naming pattern

[ObjectID]_[Method]_[YYYYMMDD]_v[##]_[stage].[ext]

What each part means

  • [ObjectID] stable ID for the scanned object or site.
  • [Method] capture or processing method (photogrammetry, lidar, mesh).
  • [YYYYMMDD] date in sortable format.
  • v[##] version number such as v01, v02, v03.
  • [stage] raw, proc, pres, access.
  • [ext] file extension for the specific output format.

Examples

PAOAN_photogrammetry_20260219_v01_raw.tif
PAOAN_mesh_20260220_v03_pres.obj
PAOAN_web_20260221_v01_access.glb

Versioning and linked files

  • Keep raw files unchanged after capture.
  • Save milestone versions during processing rather than overwriting.
  • Store linked files (geometry, materials, textures) in the same package folder.
  • Open outputs on another machine to validate that links still work.

Optional terminal templates

These can be pasted into PowerShell to create folder scaffolding quickly.

Repo-level directory template

$repo = ".\3D_Repository"
New-Item -ItemType Directory -Path $repo,(Join-Path $repo "Projects"),(Join-Path $repo "Objects"),(Join-Path $repo "Metadata"),(Join-Path $repo "Policies"),(Join-Path $repo "Resources") -Force | Out-Null
New-Item -ItemType File -Path (Join-Path $repo "Metadata\master_inventory.csv") -Force | Out-Null

Project-level directory template

$project = ".\3D_Repository\Projects\ProjectName_ObjectID"
New-Item -ItemType Directory -Path $project,(Join-Path $project "01_raw"),(Join-Path $project "02_processing"),(Join-Path $project "03_preservation"),(Join-Path $project "04_access") -Force | Out-Null
New-Item -ItemType File -Path (Join-Path $project "metadata.txt"),(Join-Path $project "paradata.txt"),(Join-Path $project "policy.txt") -Force | Out-Null
