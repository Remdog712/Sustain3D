You are on: Storage Next: Policies

Goal

Build a storage strategy that combines local, cloud, and repository options with clear governance, documentation, and sustainability planning so 3D assets remain accessible over time.

1) Storage tiers and stewardship responsibilities

Preservation is an ongoing lifecycle activity. Separate working files from preservation masters and access copies, and document who is responsible for each storage tier.

Good

  • Working, preservation, and access tiers are clearly separated.
  • One owner is assigned for storage updates.

Better

  • Tier responsibilities are documented in policy and metadata records.
  • Review dates are scheduled.

Best

  • Roles, workflows, and escalation paths are formally maintained.
  • Storage decisions are reviewed as part of routine governance cycles.

2) Local storage options

Local options include external drives, lab servers, and departmental network shares. They support active work but carry risks from media failure, obsolescence, and inconsistent backup behavior.

Good

  • Local storage is used for active work with at least one additional copy.
  • Copy destinations are documented.

Better

  • Local copies are distributed across independent systems.
  • Verification and restore checks are scheduled.

Best

  • Local storage is integrated with policy-driven redundancy.
  • Integrity monitoring and migration planning are tracked over time.

3) Cloud storage options

Cloud platforms can improve collaboration and off-site resilience, but synchronization alone is not preservation. Long-term use requires independent copies, governance controls, and exit planning.

Good

  • Cloud space is used for access and collaboration.
  • Critical files are also stored outside the sync platform.

Better

  • Cloud workflows define versioning, roles, and retention practices.
  • Platform risks and cost growth are reviewed periodically.

Best

  • Cloud storage is one layer in a multi-location preservation plan.
  • Exit strategy and migration pathways are documented in advance.

4) Repositories and persistent identifiers

Trusted repositories add structured stewardship through governance, curation standards, and long-term management. Persistent identifiers such as DOIs improve findability and citation when backed by stable organizational support.

Good

  • 3D assets are deposited in a stable repository service.
  • doi_status is tracked in project records.

Better

  • DOIs are reserved before publication.
  • Landing page and metadata links are reviewed before release.

Best

  • Repository governance and funding stability are considered in selection.
  • Identifier records stay synchronized with policy and metadata updates.

5) Local, cloud, and other storage in one plan

The most sustainable approach combines local workflow storage, independent off-site copies, and repository deposit for long-term access. Costs include storage media, subscriptions, metadata labor, review time, and migration work.

Good

  • One local working location plus one independent copy.
  • Basic documentation of responsibilities and locations.

Better

  • Local, cloud/off-site, and repository roles are clearly separated.
  • Costs and staffing needs are reviewed annually.

Best

  • Storage, governance, and funding are treated as one long-term stewardship program.
  • Planned migration and continuity workflows are maintained over time.
