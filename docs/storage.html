Goal

Place preservation masters and access files in clear storage tiers with routine copies and DOI tracking.

Keep this practical: focus on copy locations, verification, and clear documentation.

Storage tiers and what belongs in each

  • Working tier: in-progress files used for daily editing.
  • Preservation tier: preservation masters kept as controlled long-term copies.
  • Access tier: public or instructional derivatives used for delivery.
  • Print tier: optional print-ready outputs (3MF/STL/G-code) and print notes.
Apply three copies first to preservation masters, then to other tiers as capacity allows.

Storage types (practical model)

  • Local working storage for active editing and processing.
  • Preservation storage for controlled master files.
  • Off-site or independent backup for redundancy and risk reduction.
  • Repository publication storage for shared access copies and DOI landing records.

How to copy data safely

  1. Copy from working tier to preservation and backup tiers on a fixed schedule.
  2. Verify that copied files open and, when possible, run checksum comparison.
  3. Record each copy destination in project metadata and repository inventory.
  4. Do not replace preservation masters with access files.

DOI setup in storage workflow

  • Set doi_status as planned during setup.
  • Set doi_status as reserved when a repository reserves an identifier.
  • Set doi_status as published and fill doi_value after release.
  • Store the DOI landing page URL in project metadata and policy records.

Good / Better / Best

Good

  • Preservation master stored in working + one backup location.
  • DOI tracked as planned.

Better

  • Preservation master stored in working + local backup + offsite location.
  • DOI reserved before publication.

Best

  • Copy workflow is scheduled and reviewed.
  • Published DOI and access record are validated regularly.
