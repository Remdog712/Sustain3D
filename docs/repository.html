Goal

Build a repository that prevents the most common failure mode: “stuff on a drive with no context.” Your repository must keep files grouped, identifiable, and inventory-tracked.

Repository-level structure (recommended)

3D_Repository/
  Projects/
  Objects/
  Metadata/
    master_inventory.csv
  Documentation/
    naming_conventions.txt
    workflow_standards.txt
  Preservation_Logs/
    checksums/
    migration_records.txt

The “master_inventory.csv” becomes your “center database” for what exists and where it lives.

Project folder template

ProjectName_ObjectID/
  01_raw/
  02_processing/
  03_preservation/
  04_access/
  documentation/
    README.txt
    capture_notes.txt

Rules

  • Never edit inside 03_preservation/ after you finalize it.
  • Never store only a derivative (like GLB) without the preservation master.
  • Keep linked files together (OBJ + MTL + textures in one package).
