Goal

Capture enough descriptive and process context to support interpretation, reuse, and stewardship by maintaining both object-level metadata and project-level registry metadata throughout the workflow.

1) Metadata workflow and handoff

  • At setup: create project record in Metadata/project_registry.json and define workflow_doc and policy_doc.
  • After capture: record object details in object metadata text file in project documentation/.
  • During processing: update paradata notes and preservation/access file names as they are finalized.
  • After handoff: copy raw package, preservation package, and metadata into repository Preservation/<project>/<object>/.
  • During publication: update DOI status/value and policy values in project_registry.json.

3) Project registry explained + empty template

project_registry.json is the project-level control file. It tracks project identity, ownership, workflow and policy references, and publication status across all objects in a project.

  • registry_version: schema/version label for your local registry format.
  • updated: last date the registry was edited.
  • projects[]: one entry per project.
  • policy: access, rights, license, and attribution baseline for the project.
  • publication: DOI state and publication links.

  "updated": "",
  "projects": [
