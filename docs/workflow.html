PIP 0 - Repository setup and before scanning

Goal: establish structure, naming, rights, and metadata expectations before capture.

  1. Create repository and project folder structure.
  2. Assign project ID and naming pattern.
  3. Set rights and access level baseline.
  4. Define required metadata and paradata fields.

Good

  • Folders and IDs exist.
  • Basic rights note exists.

Better

  • Metadata template is prefilled.
  • FAIR and CARE notes are included.

Best

  • Team workflow standard is documented.
  • DOI plan is defined early.
After this PIP you will have: a ready repository, a project ID, and clear governance before scan day.

PIP 1 - After capture

Goal: secure raw capture data and preserve capture context.

  1. Copy raw captures into 01_raw_scans/.
  2. Record device, settings, operator, and date.
  3. Update metadata with source file references.

Good

  • Raw files copied the same day.
  • Short capture note written.

Better

  • Capture context is complete.
  • Source list is reviewed.

Best

  • Second person confirms completeness.
  • Capture protocol deviations are logged.
After this PIP you will have: preserved raw data with enough context to trust and reuse it.

PIP 2 - Processing and preservation master

Goal: create and document a stable preservation master.

  1. Save key processing milestones.
  2. Document major modeling and texturing decisions.
  3. Export a high-quality preservation master.

Good

  • Final master export is saved.
  • Basic processing notes exist.

Better

  • Milestone versions are retained.
  • Paradata includes key parameter choices.

Best

  • Full provenance trail is captured.
  • Master quality review is documented.
After this PIP you will have: a traceable preservation master that can be interpreted later.

PIP 3 - Access copies

Goal: produce delivery copies without changing preservation masters.

  1. Create access derivatives for web, teaching, or outreach.
  2. Store derivatives in a separate access folder.
  3. Link derivatives to the same object ID and metadata record.

Good

  • At least one access copy exists.
  • It is separated from the master.

Better

  • Multiple access targets are generated.
  • Metadata links each derivative.

Best

  • Access performance is tested.
  • Public description and credit text are complete.
After this PIP you will have: usable public-facing outputs that do not risk your master files.

PIP 4 - Storage, policy checks, and maintenance

Goal: place files in reliable storage tiers, validate governance, and maintain publication records.

  1. Apply the three-copy model to preservation master files.
  2. Record storage locations for working, master, and access files.
  3. Set DOI status and confirm access/policy language before release.

Good

  • Master has working and backup copies.
  • DOI planning status is tracked.

Better

  • Master has offsite copy.
  • DOI is reserved before publication.

Best

  • Storage and DOI records are reviewed regularly.
  • Access links and metadata stay synchronized.
After this PIP you will have: distributed storage for preservation masters and a clear DOI publication path.

Copy template: full setup from start to end

Run this in PowerShell. It creates repository folders, one project folder, and starter template files with content.

$repo = ".\3D_Repository"
$project = Join-Path $repo "Projects\ProjectName_ObjectID"
$docs = Join-Path $project "documentation"

New-Item -ItemType Directory -Path $repo,$project -Force | Out-Null
New-Item -ItemType Directory -Path @(
  (Join-Path $repo "Projects"),
  (Join-Path $repo "Objects"),
  (Join-Path $repo "Metadata"),
  (Join-Path $repo "Documentation"),
  (Join-Path $repo "Preservation_Logs"),
  (Join-Path $project "01_raw_scans"),
  (Join-Path $project "02_processing"),
