Workflow overview

This workflow is built around Preservation Intervention Points (PIPs) — clear moments where you stop, save the right files, and record what happened.

Core rule: treat a 3D asset as a package (model + textures + documentation), not a single file.

The workflow in one minute

  1. Plan (purpose, IDs, formats, rights).
  2. Capture and immediately save raw data + notes.
  3. Process and produce a preservation master + record decisions.
  4. Create access copies (web/teaching) separately.
  5. Protect with backups + checksums + routine checks.

PIP 0 — Before you scan

Goal: decide what you are making, what you will keep, and how you will describe it.

Do this

  1. Purpose: documentation, measurement, education, web, printing.
  2. Retention: raw capture, working files, preservation master, access derivatives.
  3. Formats: pick one preservation master + one access format.
  4. IDs: create an object/project identifier scheme.
  5. Rights: decide public / internal / restricted.

Capacity tiers

Good

  • Write a short purpose note.
  • Pick a preservation format (OBJ/PLY or E57).
  • Record rights uncertainty (if any).

Better

  • Create the project folder now.
  • Create an inventory row before capture.
  • Decide access level (public/internal/restricted).

Best

  • Write a short capture standard.
  • Confirm storage plan (3 copies from day one).

PIP 1 — Immediately after capture

Goal: don’t lose the raw data or the context while it’s still fresh.

Keep

  • Raw capture (photos, scans, point clouds, sensor exports).
  • Capture notes (device, settings, environment, operator).
  • Fixity (checksum) for the raw folder (if possible).

Capacity tiers

Good

  • Copy raw data into 01_raw/ same day.
  • Add a simple README.

Better

  • Generate checksums (SHA-256) for raw.
  • Record software + version used.

Best

  • Two-person review: confirm it opens.
  • Make raw read-only in preservation storage.
Common failure mode: raw files get moved to a drive and vanish after the project ends. Copy into the repository structure immediately.

PIP 2 — Processing and preservation master

Goal: produce a high-fidelity preservation copy and document what changed.

Do this

  1. Export milestones (aligned cloud, high-res mesh, textured mesh).
  2. Record decisions (cleanup, hole fill, smoothing, decimation, texture edits).
  3. Create a preservation master (highest fidelity you can sustain).

Capacity tiers

Good

  • Keep final best mesh + textures.
  • Write what you changed (plain language).

Better

  • Keep major milestones (v01, v02, v03).
  • Use relative paths for textures.

Best

  • Keep project files + settings.
  • Record provenance (who/when/tools).

PIP 3 — Access copies

Goal: create shareable versions without damaging or replacing the preservation masters.

Do this

  • Create access derivatives (e.g., glTF/GLB) for web/teaching.
  • Store them in 04_access/, separate from preservation.
  • Link them back to the master using the same ID + inventory record.
Rule: access formats are delivery outputs. They are not preservation masters.

PIP 4 — Storage, backups, routine checks

Goal: prevent loss, detect corruption, and be able to recover.

Minimum practice

  • 3 copies (working + local backup + offsite).
  • Fixity checks for preservation folders (on a schedule).
  • Inventory that tells you what exists and where it lives.
